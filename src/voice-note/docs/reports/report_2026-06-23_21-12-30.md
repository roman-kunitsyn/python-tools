# voice-note Recording Session Fix Report

## Summary

Fixed repeated SPACE recordings so each recording in one session writes a new
timestamped audio file. Redirected Whisper technical output to the session log.

## Changes

- Kept one session folder per `voice-note` run.
- Changed default recording output to create a fresh
  `audio/audio_{YYYY_MM_DD-HH_MM_SS}.wav` file for each SPACE recording.
- Added session `log.txt` resolution.
- Redirected `whisper-cli` stdout/stderr to `log.txt`.
- Stopped printing Whisper technical output in CLI mode.
- Changed CLI status output to `Recording...`, `Transcribe...`, then the
  transcript text without a `>` prefix.

## Checks

- `PYTHONPATH=src/voice-note uv run python -m compileall src/voice-note`
- `PYTHONPATH=src/voice-note uv run python -m unittest discover -s src/voice-note/tests`
- `uv run voice-note --help`
- `PYTHONPATH=src/voice-note uv run python -m compileall src/voice-note/voice_note src/voice-note/tests`
