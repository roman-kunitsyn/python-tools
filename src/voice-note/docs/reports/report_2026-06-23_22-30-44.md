# voice-note VS Code Link Fix Report

## Summary

Fixed transcript link behavior so VS Code mode opens `transcribe.txt` in VS
Code instead of delegating to the browser.

## Changes

- Added `TranscriptLink`, a Textual link subclass that routes click/enter to
  the app's transcript-open action.
- Changed VS Code opening to run `code -g transcribe.txt`.
- Kept nvim opening as `nvim transcribe.txt` while the TUI is suspended.
- Added editor command tests.
- Updated README and implementation plan.

## Checks

- `PYTHONPATH=src/voice-note uv run python -m compileall src/voice-note/voice_note src/voice-note/tests`
- `PYTHONPATH=src/voice-note uv run python -m unittest discover -s src/voice-note/tests`
- `PYTHONPATH=src/voice-note uv run python -c "from pathlib import Path; from voice_note.tui.app import _editor_command; print(_editor_command(Path('logs/voice_notes/session/transcribe.txt'), 'code'))"`
- Headless Textual mount check verified `#transcript-link` is a
  `TranscriptLink`.
