# voice-note Default Storage Report

## Summary

Updated `voice-note` so default runs persist audio and transcription under a
dated session folder in `logs/voice_notes`.

## Changes

- Added default session storage resolution:
  `logs/voice_notes/voice_note_{YYYY_MM_DD-HH_MM_SS}`.
- Set default audio file to `audio/audio_{timestamp}.wav`.
- Set default transcript file to `transcribe.txt`.
- Kept default recorded audio after transcription.
- Updated CLI help, README, implementation plan, and tests.

## Checks

- `PYTHONPATH=src/voice-note uv run python -m compileall src/voice-note`
- `PYTHONPATH=src/voice-note uv run python -m unittest discover -s src/voice-note/tests`
- `uv run voice-note --help`
- `uv run python -c "from voice_note.models.settings import VoiceNoteSettings; s=VoiceNoteSettings().with_default_storage(timestamp='2026_06_23-14_35_10'); print(s.audio_file); print(s.text_output_file); print(s.keep_audio)"`
