# voice-note Built-In Microphone Fix Report

## Summary

Updated `voice-note` to use the built-in microphone by default instead of the
changing macOS default input device.

## Changes

- Added `audio_device` setting with default `built-in microphone`.
- Added `--audio-device` CLI override.
- Passed the selected audio device through to `audio-record`.
- Updated macOS `audio-record` device resolution to match built-in microphone
  aliases such as `MacBook Pro Microphone`.
- Updated docs and tests.

## Local Device Check

Current FFmpeg audio devices:

- `0`: `BlackHole 2ch`
- `1`: `MacBook Pro Microphone`
- `2`: `Aggregate Device`

The `built-in microphone` alias resolves to FFmpeg input `:1`.

## Checks

- `PYTHONPATH=src/voice-note uv run python -m compileall src/voice-note src/audio-record`
- `PYTHONPATH=src/voice-note uv run python -m unittest discover -s src/voice-note/tests`
- `PYTHONPATH=src/audio-record uv run python -c "from pathlib import Path; from audio_record.models.settings import RecordingSettings; from audio_record.recorder.ffmpeg import FFmpegCommandBuilder; print(FFmpegCommandBuilder().build(RecordingSettings(output_file=Path('out.wav'), device='built-in microphone'), Path('out.wav')))"`
- `uv run voice-note --help`
