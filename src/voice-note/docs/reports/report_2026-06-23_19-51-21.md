# voice-note Implementation Report

## Summary

Implemented `voice-note` as a CLI/TUI orchestration tool for recording,
transcribing, and writing quick voice notes.

## Changes

- Added `voice-note.py` thin entry point.
- Added project packaging metadata and a `voice-note` console script.
- Added the `voice_note` package with CLI, TUI, config, models, audio,
  transcription, output, and service layers.
- Added a recorder adapter that uses the sibling `audio-record` package.
- Added a Whisper transcriber that follows the existing `whisper-cli` command
  contract.
- Added stdout and file output writers with timestamp formatting support.
- Updated default persistence to write each run under
  `logs/voice_notes/voice_note_{timestamp}`.
- Added unit tests for core service behavior and output/config helpers.

## Checks

- `PYTHONPATH=src/voice-note uv run python -m compileall src/voice-note`
- `PYTHONPATH=src/voice-note uv run python src/voice-note/voice-note.py --help`
- `PYTHONPATH=src/voice-note uv run python -m unittest discover -s src/voice-note/tests`
- `uv run python src/voice-note/voice-note.py --help`
- `uv run voice-note --help`
- `uv run python -c "from voice_note.main import main; from voice_note.tui.app import VoiceNoteApp; print(main.__name__); print(VoiceNoteApp.__name__)"`

## Notes

- CLI SPACE handling is implemented as a start/stop toggle because standard
  terminal input does not reliably expose key release events.
