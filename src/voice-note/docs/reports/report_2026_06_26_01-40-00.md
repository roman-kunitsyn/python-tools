# Report: Clarify Whisper Transcript Output Failure

## Summary

Fixed the shared Whisper transcription wrapper so a missing transcript file is
reported as a clear runtime error instead of leaking a misleading
`FileNotFoundError` from `read_text()`.

## Files Changed

- `src/voice-note/voice_note/transcription/whisper_transcriber.py`
- `src/voice-note/tests/test_voice_note_service.py`
- `src/voice-note/docs/IMPLEMENTATION_PLAN.md`
- `src/voice-note/docs/reports/report_2026_06_26_01-40-00.md`

## Behavior Changed

- `WhisperTranscriber.transcribe()` now checks that `transcript.txt` exists
  after `whisper-cli` returns.
- If the file is missing, the code raises a clear `RuntimeError` describing
  the missing output file.
- The voice-note test suite now covers the missing-output case.

## Checks Run

- `PYTHONPATH=src/voice-note uv run python -m unittest discover -s src/voice-note/tests`

## Remaining Risks

- The underlying `whisper-cli` invocation may still fail for real dependency or
  model issues.
- The runtime error message is clearer now, but the caller still needs to decide
  how to present or log it.
