# voice-note TUI Footer Status Report

## Summary

Moved TUI status display into the application footer and added background
colors for each state.

## Changes

- Removed status from the main content area.
- Added a footer status bar above the Textual key binding footer.
- Added state-specific CSS classes for idle, recording, transcribing, saved,
  and error states.
- Added helper functions for status formatting and class selection.
- Added unit coverage for status formatting/class mapping.
- Updated README and implementation plan.

## Checks

- `PYTHONPATH=src/voice-note uv run python -m compileall src/voice-note/voice_note src/voice-note/tests`
- `PYTHONPATH=src/voice-note uv run python -m unittest discover -s src/voice-note/tests`
- `uv run python -c "from voice_note.tui.app import VoiceNoteApp, _status_class; print(VoiceNoteApp.__name__); print(_status_class('Status: Recording...'))"`
- Headless Textual `run_test()` smoke check mounted `VoiceNoteApp` and verified
  the initial `status-idle` class.
