# Task Report: TUI Model File Selector

## Summary

Changed the TUI `Model File` field from a free-text path input to a dropdown populated from `~/whisper/models/`.

## Files Changed

- `src/app/models/config.py`
- `src/app/services/models.py`
- `src/app/ui/forms/transcribe_form.py`
- `README.md`
- `docs/DEVELOPMENT_GUIDELINE.md`

## Behavior Added or Changed

- Added `DEFAULT_MODEL_DIR`.
- Added `list_model_files()` for model discovery.
- Updated `TranscribeForm` to render model files as a `Select`.
- The selected model path is written back into `TranscribeConfig.model_file`.
- If no model files are found, the select shows a disabled placeholder and form submission raises a validation error.

## Checks Run

- `uv run python -m py_compile src/app/models/config.py src/app/services/models.py src/app/ui/forms/transcribe_form.py`
- `uv run python -c "from src.app.services.models import list_model_files; print([p.name for p in list_model_files()])"`
- Textual mounted form smoke test for model select value.
- Textual mounted form smoke test for `TranscribeConfig.model_file`.

## Remaining Risks

- The dropdown currently lists all regular files in `~/whisper/models/`; it does not filter by extension.
