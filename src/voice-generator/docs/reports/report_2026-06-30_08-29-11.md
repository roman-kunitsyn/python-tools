# voice-generator Orpheus SNAC Decoder Fix

## Summary

Replaced the synthetic SNAC placeholder decoder with a real codec-based decode
path and kept the official-style Orpheus prompt/token extraction flow.

## What Changed

- Removed the fake sine-wave decoder logic.
- Added real SNAC model loading behind a lazy runtime boundary.
- Reconstructed the hierarchical SNAC codebooks from the emitted token stream.
- Decoded the resulting audio with the SNAC codec instead of a secondary TTS
  engine.
- Kept WAV as the output artifact and preserved format conversion through the
  shared audio utility.
- Updated tests to mock the SNAC codec model instead of relying on the fake
  tone generator.

## Validation

- `uv run python -m compileall src/voice-generator/voice_generator src/voice-generator/tests`
- `uv run python -m unittest discover -s src/voice-generator/tests`

## Notes

- The decoder now requires the `snac` and `torch` packages at runtime.
- The Orpheus provider still owns the complete synthesis pipeline.
