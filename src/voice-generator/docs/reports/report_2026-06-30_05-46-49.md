# voice-generator WAV-First Output Update

## Summary

Updated the macOS provider to treat WAV as the standard output format while
keeping AIFF available as an explicit option. The docs were updated to match.

## Files Changed

- `src/voice-generator/README.md`
- `src/voice-generator/docs/DEVELOPMENT_GUIDELINE.md`
- `src/voice-generator/docs/IMPLEMENTATION_PLAN.md`
- `src/voice-generator/docs/reports/report_2026-06-30_05-46-49.md`
- `src/voice-generator/tests/test_macos_say.py`
- `src/voice-generator/voice_generator/providers/macos_say.py`

## Behavior Changed

- `voice-generator generate --provider macos` now standardizes on WAV output.
- The macOS backend still uses `say` to synthesize an AIFF intermediate file
  and converts it to WAV with ffmpeg when needed.
- AIFF output remains supported when the user explicitly requests it.
- The README and development guideline now describe WAV as the default output
  format.

## Checks Run

- `uv run python -m compileall src/voice-generator/voice_generator src/voice-generator/tests`
- `uv run python -m unittest discover -s src/voice-generator/tests`
- `uv run voice-generator generate --provider macos --voice Samantha --text "Hello Roman" --output /tmp/voice-generator-macos-test.wav`
- `file /tmp/voice-generator-macos-test.wav`

## Remaining Risks

- WAV output still depends on ffmpeg for the conversion step after native
  synthesis.
- Orpheus remains adapter-driven and still requires a user-provided runtime
  command and model path.
