# voice-generator First Implementation Slice

## Summary

Started the `voice-generator` implementation with a runnable package scaffold,
shared models, a provider catalog, config loading, and a thin CLI boundary.

## Behavior Added

- Added a `voice-generator` script entry point and a direct script at
  `src/voice-generator/voice-generator.py`.
- Added `voice providers` output with the planned provider catalog.
- Added `voice voices` scaffold output for the provider catalog boundary.
- Added `voice validate` with `ffmpeg`, directory, default-provider, and
  ElevenLabs API key checks.
- Added request, response, provider metadata, and voice metadata models.
- Added placeholder generation and benchmark service boundaries.
- Added a flat config loader that accepts simple `key: value` files.

## Checks Run

- `uv run python -m compileall src/voice-generator/voice_generator src/voice-generator/tests`
- `uv run python -m unittest discover -s src/voice-generator/tests`
- `uv run python src/voice-generator/voice-generator.py providers`
- `uv run python src/voice-generator/voice-generator.py validate`
- `uv run python src/voice-generator/voice-generator.py --help`
- `uv sync --reinstall`
- `uv run voice-generator --help`
- `uv run voice-generator providers`
- `uv run voice-note --help`

## Remaining Risks

- Provider backends are still stubbed.
- Real generation, benchmarking, and voice discovery are not implemented yet.
- The config loader currently supports a simple flat mapping format, not a full
  YAML parser.
- Generation and benchmark commands intentionally report the feature as not
  implemented.
