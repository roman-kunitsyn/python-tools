# voice-generator Orpheus Prompt and Token Extraction

## Summary

Updated the Orpheus `llama.cpp` runtime to use a chat-style prompt that forces
audio-token generation and extracts `<custom_token_xxxxx>` output before SNAC
decoding.

## What Changed

- Added an Orpheus prompt with explicit system and user headers.
- Added the assistant-side generation prefix used before audio token output.
- Enabled llama.cpp special-token output so custom tokens are surfaced in the
  runtime stream.
- Extracted `<custom_token_xxxxx>` tokens from the runtime output.
- Passed token sequences directly to the SNAC decoder.
- Stored extracted tokens in the returned `AudioResult.metadata`.
- Added tests for token extraction from stdout and stderr.

## Validation

- `uv run python -m compileall src/voice-generator/voice_generator src/voice-generator/tests`
- `uv run python -m unittest discover -s src/voice-generator/tests`

## Notes

- This slice keeps the Orpheus synthesis path self-contained.
- No secondary TTS engine is involved.
