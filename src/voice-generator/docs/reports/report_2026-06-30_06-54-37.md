# voice-generator Orpheus Runtime Refactor

## Summary

Refactored the Orpheus provider from a split text/audio subprocess pipeline to
an internal runtime-driven architecture with a decoder boundary.

## What Changed

- Removed the public Orpheus stage flags for text and audio subprocess hooks.
- Added Orpheus runtime selection via `runtime`, `model`, `executable`, and
  `decoder`.
- Introduced internal runtime modules under `voice_generator/runtimes/`.
- Introduced an internal SNAC decoder under `voice_generator/decoders/`.
- Updated the public provider to expose `synthesize()` and keep `generate()`
  as a compatibility alias.
- Updated config loading to support the new nested `orpheus:` layout while
  preserving compatibility aliases for older flat config keys.
- Updated CLI help, README guidance, and implementation documentation.

## Validation

- `uv run python -m compileall src/voice-generator/voice_generator src/voice-generator/tests`
- `uv run python -m unittest discover -s src/voice-generator/tests`

## Notes

- Orpheus runtime selection is now explicit.
- Provider status reports runtime, model, executable, and decoder readiness.
- `VoiceResponse` remains available as a compatibility alias for
  `AudioResult`.
