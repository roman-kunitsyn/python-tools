# voice-generator Real Backend Slice

## Summary

Implemented the first real provider backends for `voice-generator`:
macOS Say is now a live synthesis backend, and Orpheus is now a configurable
runtime adapter with model and voice catalog support.

## Behavior Added

- `macos` provider now enumerates real voices from `say -v ?`.
- `macos` provider now synthesizes audio through `say` and writes AIFF or
  converted output files.
- `voice-generator generate --provider macos` now produces a real audio file.
- `voice-generator providers` now reports `macos` as `ready` on systems with
  `say` installed.
- `orpheus` provider now validates a runtime command and GGUF model path.
- `orpheus` provider now supports voice catalog loading from JSON or newline
  files.
- `voice-generator generate --provider orpheus` now executes a configured
  runtime command template and writes generated audio through the shared
  pipeline.
- Added CLI overrides for Orpheus runtime command, model path, voice catalog,
  and command template.
- Added backend tests for voice parsing, generation command handling, and
  catalog loading.

## Remaining Risks

- Orpheus still depends on a user-supplied runtime command and model path.
- The exact Orpheus prompt and SNAC pipeline are adapter-driven rather than
  hard-coded to a single upstream runtime.
- Kokoro, Piper, and ElevenLabs are still not implemented.
- Benchmark execution is still a stub.
