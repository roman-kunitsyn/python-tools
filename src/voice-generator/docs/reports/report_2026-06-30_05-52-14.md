# voice-generator Default Output Path Update

## Summary

Changed the default no-output-path behavior so generated audio now lands in a
timestamped WAV file under `./logs/voice-generator/`.

## Files Changed

- `src/voice-generator/README.md`
- `src/voice-generator/docs/DEVELOPMENT_GUIDELINE.md`
- `src/voice-generator/docs/IMPLEMENTATION_PLAN.md`
- `src/voice-generator/docs/reports/report_2026-06-30_05-52-14.md`
- `src/voice-generator/tests/test_audio_utils.py`
- `src/voice-generator/voice_generator/utils/audio.py`

## Behavior Changed

- If the user does not pass `--output`, the tool now writes to
  `./logs/voice-generator/{YYYY_MM_DD-HH_MM_SS}.wav`.
- The default output file is created relative to the current working directory.
- The path is created automatically before synthesis writes the file.

## Checks Run

- `uv run python -m compileall src/voice-generator/voice_generator src/voice-generator/tests`
- `uv run python -m unittest discover -s src/voice-generator/tests`
- `uv run voice-generator generate --provider macos --voice Samantha --text "Hello Roman"`
- `ls -lh logs/voice-generator/2026_06_30-05_52_00.wav`
- `file logs/voice-generator/2026_06_30-05_52_00.wav`

## Remaining Risks

- The default output file is timestamp-based, so repeated runs create separate
  files rather than overwriting a single location.
- WAV output still depends on ffmpeg for the macOS `say` backend conversion
  path.
