# Report: Rewrite Markdown Image References to Local Paths

## Summary

Updated crawl-to-Markdown export so image references in generated Markdown are
rewritten to the local `images/` folder inside the session tree.

## Files Changed

- `src/browser-automation/browser_automation/exporter.py`
- `src/browser-automation/tests/test_browser_automation.py`
- `src/browser-automation/README.md`
- `src/browser-automation/docs/IMPLEMENTATION_PLAN.md`
- `src/browser-automation/docs/reports/report_2026-06-28_12-27-09.md`

## Behavior Changed

- Generated Markdown now points image references at downloaded local files.
- Remote image URLs are no longer left in the final Markdown output when a
  corresponding local download succeeded.
- Existing per-page manifests and session-level image indexing remain intact.

## Checks Run

- `PYTHONPATH=src/browser-automation python -m unittest discover -s src/browser-automation/tests`
- `python -m compileall src/browser-automation/browser_automation src/browser-automation/tests`

## Remaining Risks

- If an image download fails, its remote reference may remain in the Markdown
  because there is no local file to point at.
- The current rewrite helper uses exact URL replacement, so any future content
  transformations that alter the embedded URL string may need a more structured
  parser.
