# Report: Normalize Markdown Image Rewriting

## Summary

Fixed crawl Markdown image rewriting so protocol-relative URLs are rewritten to
local files and transparent placeholder GIF images are filtered out of the
export.

## Files Changed

- `src/browser-automation/browser_automation/exporter.py`
- `src/browser-automation/tests/test_browser_automation.py`
- `src/browser-automation/README.md`
- `src/browser-automation/docs/IMPLEMENTATION_PLAN.md`
- `src/browser-automation/docs/reports/report_2026-06-28_12-35-49.md`

## Behavior Changed

- Markdown image references now normalize against the page URL before being
  rewritten.
- Protocol-relative image URLs like `//img1.wsimg.com/...` are now rewritten to
  the matching local file path.
- Transparent placeholder GIF images are filtered out of crawl exports so they
  do not appear as dirty links in the generated Markdown.

## Checks Run

- `PYTHONPATH=src/browser-automation python -m unittest discover -s src/browser-automation/tests`
- `python -m compileall src/browser-automation/browser_automation src/browser-automation/tests`

## Remaining Risks

- Any image URL that is transformed during rendering in a way that no longer
  resembles the source URL may still need a more semantic mapping approach.
- Some pages may emit placeholder images that are not the common transparent GIF
  pattern, which would need additional filtering rules if they become a problem.
