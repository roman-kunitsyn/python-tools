# Report: Fix Crawled Image Extensions

## Summary

Fixed crawl Markdown image downloads so saved images use `.png` or `.jpg`
extensions instead of the placeholder `.img` fallback.

## Files Changed

- `src/browser-automation/browser_automation/exporter.py`
- `src/browser-automation/tests/test_browser_automation.py`
- `src/browser-automation/README.md`
- `src/browser-automation/docs/IMPLEMENTATION_PLAN.md`
- `src/browser-automation/docs/reports/report_2026-06-28_11-30-07.md`

## Behavior Changed

- Image downloads now infer an extension from the source URL or response
  content type.
- The default fallback extension is now `.png`.
- JPEG sources normalize to `.jpg`.

## Checks Run

- `PYTHONPATH=src/browser-automation python -m unittest discover -s src/browser-automation/tests`
- `python -m compileall src/browser-automation/browser_automation src/browser-automation/tests`

## Remaining Risks

- Some servers may return generic content types or no extension, so filename
  inference still falls back to `.png`.
- The downloader still fetches images directly from remote URLs during crawl
  export.
