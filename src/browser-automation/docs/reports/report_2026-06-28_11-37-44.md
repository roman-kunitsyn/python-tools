# Report: Broaden Image Discovery for Crawl Markdown

## Summary

Expanded image discovery so crawl Markdown exports collect images from lazy
loaded attributes, `srcset`, background-image URLs, and external domains.

## Files Changed

- `src/browser-automation/browser_automation/extractors/images.py`
- `src/browser-automation/tests/test_browser_automation.py`
- `src/browser-automation/README.md`
- `src/browser-automation/docs/IMPLEMENTATION_PLAN.md`
- `src/browser-automation/docs/reports/report_2026-06-28_11-37-44.md`

## Behavior Changed

- Image extraction now includes:
  - `src`
  - `data-src`
  - `data-lazy-src`
  - `data-original`
  - `data-srcset`
  - `srcset`
  - inline `background-image` URLs
- External image URLs are no longer filtered out during crawl export.

## Checks Run

- `PYTHONPATH=src/browser-automation python -m unittest discover -s src/browser-automation/tests`
- `python -m compileall src/browser-automation/browser_automation src/browser-automation/tests`

## Remaining Risks

- Some pages hide image URLs in JavaScript state rather than HTML attributes,
  which this extractor still does not parse.
- Very complex responsive image markup can still produce duplicate or redundant
  URLs, though the extractor now de-duplicates exact matches.
