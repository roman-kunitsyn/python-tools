# Report: Harden Crawl Image Downloads

## Summary

Prevented crawl Markdown image downloads from failing on malformed or
non-HTTP image URLs and improved extension detection for transformed image
URLs.

## Files Changed

- `src/browser-automation/browser_automation/exporter.py`
- `src/browser-automation/tests/test_browser_automation.py`
- `src/browser-automation/docs/reports/report_2026-06-28_11-40-36.md`

## Behavior Changed

- Invalid image URLs are now skipped instead of aborting the whole crawl.
- Transformed image URLs such as `...jpeg/:/cr=...` now resolve to `.jpg`.
- Crawl Markdown export continues saving valid external images from other
  domains.

## Checks Run

- `PYTHONPATH=src/browser-automation python -m unittest discover -s src/browser-automation/tests`
- `python -m compileall src/browser-automation/browser_automation src/browser-automation/tests`

## Remaining Risks

- Pages that hide images only in JavaScript state can still be missed until a
  browser-DOM-based extractor is added.
- Some malformed image URLs may still be skipped if the source markup is
  unusable.
