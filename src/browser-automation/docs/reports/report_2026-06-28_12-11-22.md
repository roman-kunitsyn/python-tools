# Report: Improve Rendered Image Collection

## Summary

Fixed image collection for lazy-loaded and transformed images by adding a
best-effort scroll pass, using browser-resolved `currentSrc`, and waiting for
page load to settle before extraction.

## Files Changed

- `src/browser-automation/browser_automation/browser.py`
- `src/browser-automation/browser_automation/crawler.py`
- `src/browser-automation/browser_automation/extractors/images.py`
- `src/browser-automation/README.md`
- `src/browser-automation/docs/IMPLEMENTATION_PLAN.md`
- `src/browser-automation/docs/reports/report_2026-06-28_12-11-22.md`

## Behavior Changed

- Browser sessions now wait briefly for network idle after navigation.
- Crawler runs a best-effort scroll pass before collecting live images.
- Live image extraction prefers browser-resolved `currentSrc`.
- The crawler still falls back to HTML extraction when needed.

## Checks Run

- `PYTHONPATH=src/browser-automation python -m unittest discover -s src/browser-automation/tests`
- `python -m compileall src/browser-automation/browser_automation src/browser-automation/tests`

## Remaining Risks

- Pages that build image URLs only in JavaScript data structures and never in
  DOM attributes or computed styles can still be missed.
- The auto-scroll pass is best effort and may not trigger every lazy-load
  implementation on the web.
