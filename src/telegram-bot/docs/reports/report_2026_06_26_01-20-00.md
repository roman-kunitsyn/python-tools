# Report: Voice-Note Session Workflow

## Summary

Updated `telegram-bot` to use a session-based voice-note flow with Aiogram 3
FSM state, inline actions, local storage under `logs/voice_notes`, WAV
conversion before transcription, and structured service-layer logging.

## Behavior Added

- `/start`, `/help`, `/menu`, `/voice_note`, and `/cancel`
- session lifecycle with `waiting_voice` and `waiting_decision` states
- multiple recordings per session
- local storage for each uploaded file and its WAV conversion
- automatic transcription per recording
- inline `Add More`, `Finish`, and `Cancel` actions
- transcript and metadata files per session
- cancel cleanup that removes temporary files
