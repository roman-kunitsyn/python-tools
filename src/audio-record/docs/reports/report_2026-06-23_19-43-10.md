# audio-record Implementation Report

## Summary

Implemented the reusable `audio-record` tool as a focused FFmpeg wrapper for
recording audio to a file.

## Changes

- Added a thin CLI script at `audio-record.py`.
- Added the `audio_record` package with CLI parsing, dataclass settings,
  device models, FFmpeg command construction, subprocess recording, and temp
  file utilities.
- Added the public `AudioRecorder` API with `record()`, `start()`, and
  `stop()`.
- Added `get_audio_devices()` and `AudioDevice` for device discovery.
- Added module development docs and implementation plan.

## Checks

- `uv run python -m compileall src/audio-record`
- `uv run python src/audio-record/audio-record.py --help`
- `PYTHONPATH=src/audio-record uv run python -c "from audio_record import AudioRecorder, get_audio_devices; print(AudioRecorder.__name__); print(callable(get_audio_devices))"`
- `PYTHONPATH=src/audio-record uv run python -c "from pathlib import Path; from audio_record.models.settings import RecordingSettings; from audio_record.recorder.ffmpeg import FFmpegCommandBuilder; print(FFmpegCommandBuilder().build(RecordingSettings(duration=1), Path('out.wav')))"`

## Risks

- Device discovery output varies by FFmpeg version and host audio backend.
- Linux support currently targets PulseAudio for recording and discovery.
