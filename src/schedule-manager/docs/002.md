# The most important design decision

Do not build:

```text
Textual App
```

Build:

```text
Planner Engine
```

and then

```text
Textual UI
```

on top.

A good test is:

```python
planner = Planner()

planner.create_event(...)
planner.create_note(...)
planner.create_task(...)
```

should work completely without Textual.

If that's true, you have a production-quality architecture.

Textual then becomes only one presentation layer. Later you could add:

```text
CLI
REST API
Web UI
Mobile App
```

without touching the business logic. That's the mindset experienced Textual developers use when they expect a project to grow beyond a toy application.
